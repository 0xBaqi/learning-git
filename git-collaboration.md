\# Git Collaboration



\## Working with a Team



\### Cloning a shared repo

\- git clone <url>



\### Keeping up to date

\- git pull origin main

\- git fetch --all



\### Working on features

1\. git switch -c feature/name

2\. Make changes

3\. git add .

4\. git commit -m "message"

5\. git push origin feature/name

6\. Open pull request on GitHub



\### Code Review

\- Team members review your pull request

\- They can leave comments and suggestions

\- Make changes based on feedback

\- Push new commits to same branch

\- PR updates automatically



\### Merging

\- Only merge after approval

\- Delete branch after merging

\- Pull latest main after merge



\## Best Practices

\- Never force push to main

\- Always pull before starting work

\- Keep pull requests small and focused

\- Write clear commit messages

\- Communicate with your team

\- Review others code promptly

\- Test before merging



\## Handling Conflicts in a Team

1\. git pull origin main

2\. Resolve conflicts locally

3\. git add .

4\. git commit -m "resolve conflicts"

5\. git push origin feature/name
