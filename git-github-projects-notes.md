\# GitHub Projects Notes



\## What are GitHub Projects?

GitHub Projects is a project management tool

built into GitHub. It lets you organize issues

pull requests and tasks into boards timelines

and tables for better workflow management.



\## Types of Project Views



\### Board View

\- Kanban style columns

\- Drag and drop cards

\- Visual workflow management

\- Great for tracking progress



\### Table View

\- Spreadsheet like layout

\- Sort and filter items

\- Add custom fields

\- Good for detailed tracking



\### Timeline View

\- Gantt chart style

\- Date based planning

\- See dependencies

\- Good for release planning



\## Creating a Project

1\. Go to your GitHub profile or organization

2\. Click Projects tab

3\. Click New project

4\. Choose template or start blank

5\. Name your project

6\. Add views and fields



\## Adding Items to Project

\- Add existing issues and pull requests

\- Create new draft issues directly

\- Convert drafts to real issues

\- Link items from multiple repos



\## Custom Fields

\- Text — notes and descriptions

\- Number — story points or estimates

\- Date — due dates and milestones

\- Single select — status priority labels

\- Iteration — sprint planning



\## Common Project Templates



\### Team Backlog

\- Backlog column

\- In Progress column

\- In Review column

\- Done column



\### Bug Tracker

\- New bugs column

\- Investigating column

\- In Progress column

\- Fixed column



\### Feature Planning

\- Ideas column

\- Planned column

\- In Development column

\- Released column



\## Automations

\- Auto add new issues to project

\- Auto move to In Progress when assigned

\- Auto move to Done when closed

\- Reduces manual project management



\## Workflows with Projects



\### Sprint Planning

1\. Add all planned issues to project

2\. Set iteration field for sprint

3\. Assign to team members

4\. Track progress on board



\### Release Planning

1\. Create milestone on GitHub

2\. Add all release issues to project

3\. Use timeline view for scheduling

4\. Track completion towards release



\## Linking Projects to Repos

\- Projects can span multiple repos

\- Great for large team projects

\- Single view of all work

\- Filter by repo when needed



\## GitHub Projects vs Issues

\- Issues — individual tasks and bugs

\- Projects — organize and track issues

\- Projects give the big picture view

\- Issues have the detailed information



\## Tips

\- Start with simple board view

\- Add custom fields as needs grow

\- Use automations to reduce manual work

\- Review project board in team meetings

\- Archive completed iterations regularly

\- Link every PR to an issue in project
