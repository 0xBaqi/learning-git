\# GitHub Pages Notes



\## What is GitHub Pages?

GitHub Pages is a free hosting service from GitHub.

It lets you host static websites directly from a repo.



\## What You Can Host

\- Personal portfolio websites

\- Project documentation

\- Blog with Jekyll

\- Landing pages

\- HTML CSS JS projects



\## Setting Up GitHub Pages



\### Method 1 - From main branch

1\. Go to repo on GitHub

2\. Click Settings

3\. Click Pages in sidebar

4\. Select main branch as source

5\. Click Save

6\. Your site is live at username.github.io/repo



\### Method 2 - From docs folder

1\. Create a docs folder in your repo

2\. Add your HTML files inside

3\. Go to Settings > Pages

4\. Select main branch and /docs folder

5\. Click Save



\### Method 3 - From gh-pages branch

1\. Create a gh-pages branch

2\. Add your HTML files

3\. Push to GitHub

4\. Pages auto detects gh-pages branch



\## Custom Domain

1\. Buy a domain from any registrar

2\. Go to Settings > Pages

3\. Add your custom domain

4\. Update DNS records with your registrar

5\. Enable HTTPS



\## Jekyll

\- Static site generator supported by GitHub Pages

\- Write content in Markdown

\- Automatically converts to HTML

\- Many free themes available



\## Tips

\- Site may take a few minutes to go live

\- Clear browser cache if changes dont show

\- Use relative paths for links and images

\- Test locally before pushing

\- README.md shows as homepage if no index.html



\## Limitations

\- Static sites only, no server side code

\- 1GB storage limit

\- 100GB bandwidth per month

\- No private page hosting on free plan
