\# GitHub Issues Notes



\## What are GitHub Issues?

Issues are used to track bugs, feature requests,

and tasks in a GitHub repository.

They are the main way to communicate about work.



\## Creating an Issue

1\. Go to repo on GitHub

2\. Click Issues tab

3\. Click New Issue

4\. Add title and description

5\. Assign labels, milestone, assignees

6\. Click Submit new issue



\## Issue Components

\- Title — short summary of the issue

\- Description — detailed explanation

\- Labels — categorize the issue

\- Assignees — who is working on it

\- Milestone — which version it belongs to

\- Comments — discussion thread



\## Common Labels

\- bug — something is broken

\- feature — new functionality request

\- documentation — docs need updating

\- help wanted — open for contributors

\- good first issue — good for beginners

\- duplicate — already reported

\- wontfix — will not be addressed



\## Linking Issues to Commits

\- Add issue number in commit message

\- git commit -m "fix login bug closes #42"

\- GitHub auto closes issue when merged



\## Closing Keywords

\- fixes #42

\- closes #42

\- resolves #42

\- Use in commit message or PR description



\## Issue Templates

\- Create templates for bug reports

\- Create templates for feature requests

\- Stored in .github/ISSUE\_TEMPLATE/

\- Helps contributors provide right info



\## Best Practices

\- Search before creating new issue

\- One issue per bug or feature

\- Be specific and detailed

\- Add screenshots when relevant

\- Respond to comments promptly

\- Close issues when resolved

\- Use milestones to track progress



\## Using Issues for Project Management

\- Create issues for all tasks

\- Use labels to organize

\- Use milestones for versions

\- Use GitHub Projects for kanban board

\- Link PRs to issues
