\# Git Workflow



\## Daily Workflow

1\. git pull — get latest changes

2\. git switch -c feature/name — create new branch

3\. Make changes to files

4\. git add . — stage changes

5\. git commit -m "message" — commit changes

6\. git push origin feature/name — push to GitHub

7\. Create pull request on GitHub

8\. Merge pull request

9\. git switch main — go back to main

10\. git pull — update local main

11\. git branch -d feature/name — delete local branch



\## Commit Message Tips

\- Use present tense: "add feature" not "added feature"

\- Be specific: "fix login bug" not "fix bug"

\- Keep it short and clear



\## Branch Naming

\- feature/feature-name

\- fix/bug-name

\- docs/update-readme

\- hotfix/critical-fix



\## Good Habits

\- Commit often, push daily

\- Never commit directly to main

\- Always pull before starting work

\- Write meaningful commit messages

\- Keep branches short-lived
